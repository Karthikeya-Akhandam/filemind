# Security Policy

## Reporting a Vulnerability

The FileMind project takes the security of its users and data very seriously. We appreciate your efforts to responsibly disclose any vulnerabilities you may find.

If you discover a security vulnerability within FileMind, please report it immediately by sending an confidential email to [YOUR_SECURITY_EMAIL@example.com](mailto:YOUR_SECURITY_EMAIL@example.com).

Please include:
- A detailed description of the vulnerability.
- Steps to reproduce the vulnerability (if applicable).
- The affected version(s) of FileMind.
- Any potential impact or exploits.

Please **do not** open a public GitHub issue for security vulnerabilities. We will acknowledge your report within [e.g., 24-48 hours] and will work with you to understand and resolve the issue.

## Our Commitment

We are committed to:
- Promptly investigating all reported vulnerabilities.
- Keeping you informed about the progress of your report.
- Working to release fixes in a timely manner.
- Crediting responsible disclosures, if you agree.

## Supported Versions

At present, only the latest stable release of FileMind is actively supported with security updates. We encourage all users to update to the newest version available.

## Responsible Disclosure

We kindly request that you:
- Do not publicly disclose the vulnerability until it has been patched.
- Give us a reasonable amount of time to address the issue before public disclosure.
